# pc2web: Guía técnica interna

## Propósito

Desarrollar una herramienta modular en PHP + JavaScript que transforme
capas QGIS (GeoJSON + estilo + metadatos) en proyectos web
autosuficientes listos para publicar, sin necesidad de XAMPP ni
servidores adicionales.

## Arquitectura general

El sistema puede ejecutarse directamente con el servidor embebido de PHP
(`php -S localhost:8000`) o dentro de XAMPP. No requiere conexión a base
de datos.

### Objetivo técnico

Generar un entorno dual: - **Modo Productor:** permite cargar capas,
estilos y metadatos, previsualizarlos con Leaflet y exportar el
resultado como paquete web. - **Modo Visor:** muestra la versión final
exportada, lista para publicación.

pc2web/ ├── index.php ├── config/ │ ├── settings.yaml │ └── config.php
├── modules/ │ ├── navbar.php │ ├── sidebar.php │ ├── map_container.php
│ ├── right_panel.php │ ├── footer.php │ └── uploader.php ├── assets/ │
├── css/ │ ├── js/ │ └── img/ ├── data/ │ ├── input/ │ └── cache/ ├──
build/ │ └── export/ ├── logs/ ├── docs/ ├── tests/ └── proyecto.md

## Librerías principales

-   PHP \>= 8.1
-   JavaScript ES6
-   Leaflet 1.9.x
-   Bootstrap 5.x

## Flujo técnico

1.  Carga de insumos desde `/data/input/`.
2.  Validación de estructura y metadatos.
3.  Combinación con plantillas PHP/HTML en `/modules/`.
4.  Generación de salida autosuficiente en `/build/export/`.
5.  Registro de logs en `/logs/`.

## Buenas prácticas

-   Estructura modular y desacoplada.
-   Código documentado y versionado.
-   Configuración centralizada en YAML.
-   Compatibilidad multiplataforma.
